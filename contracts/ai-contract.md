# AI Contract

Claude and OpenAI are server-side only through trusted backend code such as Python FastAPI services or Supabase Edge Functions.

## Allowed AI Uses

Claude/OpenAI may:

- create validated structured query plans
- extract draft policy rules from pasted policy text
- explain returned spend results
- explain deterministic policy outcomes
- recommend approval decisions from supplied facts
- summarize generated reports
- identify missing context for readiness fields

OpenAI embeddings may:

- embed policy chunks
- embed policy retrieval queries
- support policy RAG retrieval

## Prohibited AI Uses

Claude must not:

- invent transaction, employee, department, approval, receipt, budget, or report data
- invent totals
- invent policy outcomes
- invent risk scores
- activate policy rules without manager review
- decide compliance during full scans
- override deterministic policy or risk results

Talk to Data SQL exception:

- Claude may propose read-only SQL for ad hoc analytics only inside the backend Talk to Data workflow.
- Proposed SQL must pass a second AI validation step, configured to use Claude Haiku 4.5, plus deterministic backend safety checks before execution.
- The backend may execute only a single read-only query or CTE chain inside a read-only transaction with restricted schema context and row limits.
- The frontend must never execute or assemble LLM-generated SQL directly.

OpenAI embeddings must not:

- calculate spend totals
- enforce policy thresholds
- detect duplicates or split transactions
- score risk

## Edge Function Boundary

Frontend React code may read only:

- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_ANON_KEY`

Server-only values such as `ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, `SUPABASE_SERVICE_ROLE_KEY`, and `SUPABASE_DB_URL` must never be imported, logged, or referenced in frontend code.

## Policy Rule Extraction Shape

Policy rule extraction receives pasted policy text, company context, and the backend allowlist of available fields/operators. The model must return strict JSON containing draft rules, ambiguities, unsupported or missing fields, suggested feature-engineering ideas, and a summary.

Each extracted draft rule should use canonical nested `rule_json` with:

- `condition`
- `outcome`
- `scope`
- optional `applies_to`
- optional `context_requirements`
- optional `requires`

The extraction contract is deliberately domain-neutral. The model should map source-policy facts into declarative rules and must not assume Brim-specific reimbursement workflows unless the source text and available fields explicitly support them.

Every AI-extracted rule must be validated by backend code and saved as draft with `needs_human_review = true`. Invalid or unsupported rules should be reported to the user instead of being coerced into active policy.

## Query Planner Shape

Claude returns JSON compatible with `QueryPlan` from `src/types/domain.ts`. The app validates the plan before translating it to safe Supabase query builder calls.

## RAG Explanation Shape

Policy explanation calls receive:

- deterministic policy check result
- violations and missing information
- retrieved policy chunks
- transaction facts

The model returns plain-English explanation and next steps. It does not alter persisted compliance status.

## Approval Recommendation Shape

Approval recommendations may be generated by the backend deterministic fallback or by OpenAI structured output when `OPENAI_API_KEY` is configured. OpenAI recommendations receive only the saved approval context plus deterministic fallback facts, and backend sanitization prevents an AI approval from overriding deterministic deny/readiness constraints.

Approval recommendations return:

- `recommendation: approve | deny`
- `confidence: low | medium | high`
- `rationale`
- `groundedInputs`
- `missingInformation`
- `source: deterministic_fallback | openai_structured_output`

Recommendations are advisory and must be saved with an audit trail when used. Missing information may reduce readiness or confidence, but it is not an approval recommendation or human decision outcome.
