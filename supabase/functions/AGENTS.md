# supabase/functions/AGENTS.md

This folder contains Supabase Edge Functions for secure server-side operations and compatibility workflows.

FastAPI is now the primary backend boundary for business logic. Add or change Edge Functions only when the work is specifically function-oriented or the user asks for it.

## Responsibilities

- Call AI services for compatibility query planning, approval recommendations, report summaries, and policy explanations only when the workflow has not moved fully behind FastAPI.
- Call OpenAI for policy embeddings.
- Use Supabase service role only when needed for privileged server-side reads/writes.
- Return strict JSON responses to the frontend.

## Function design rules

- Validate input payloads before using them.
- Return predictable JSON shapes.
- Keep CORS handling centralized in `_shared/cors.ts`.
- Keep Claude client setup centralized in `_shared/claudeClient.ts`.
- Keep OpenAI embedding client setup centralized in `_shared/openaiClient.ts`.
- Keep Supabase admin client setup centralized in `_shared/supabaseAdmin.ts`.

## AI safety rules

- Claude must not invent transaction totals, employee names, departments, or policy outcomes.
- Claude may only summarize or recommend based on tool/database outputs.
- Approval recommendations are approve/deny only; missing context belongs in readiness/context fields.
- If context is missing, return missing fields rather than hallucinating.
- Never execute SQL generated by Claude.

## Recommended functions

- `ai-query-planner`
- `ai-approval-recommendation`
- `ai-report-summary`
- `ai-policy-explanation`
- `embed-policy-chunks`
- `retrieve-policy-clauses`
